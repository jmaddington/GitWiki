Setup Instructions
1. SSH Key Setup

Generate an SSH key pair if you don't have one:

ssh-keygen -t ed25519 -C "gitwiki@example.com"

Add the public key to GitHub:

  • Go to GitHub → Settings → SSH and GPG keys → New SSH key
  • Copy your public key: cat ~/.ssh/id_ed25519.pub
  • Paste and save
2. GitHub Repository Setup
  1. Create a GitHub repository (or use existing)
  2. Copy the SSH URL: git@github.com:username/repository.git
  3. Paste it in the "GitHub Remote URL" field above
  4. Click "Test SSH Connection" to verify
  5. Save settings
3. Webhook Setup (Optional)

Configure GitHub to notify GitWiki when changes are pushed:

  1. Go to your GitHub repository → Settings → Webhooks → Add webhook
  2. Payload URL: https://your-domain.com/api/git/webhook/
  3. Content type: application/json
  4. Secret: Enter a random string (copy it to "Webhook Secret" field above)
  5. Events: Select "Just the push event"
  6. Active: ✓ Check
  7. Add webhook
4. Testing

After configuration, test the integration:

  • Click "Test SSH Connection" on this page
  • Go to Sync Management
  • Click "Pull from GitHub" to test sync
5. Webhook Endpoint

Your webhook endpoint URL:

{{ request.scheme }}://{{ request.get_host }}/api/git/webhook/

Rate limited to 1 pull per minute. Returns HTTP 429 if exceeded.

Security Notice
  • SSH Keys: Store your private key securely. Never commit it to the repository.
  • Webhook Secret: Use a strong, random secret for webhook signature verification.
  • HTTPS: Use HTTPS for the webhook endpoint in production.
  • Permissions: Only grant GitWiki access to specific repositories (use deploy keys).
